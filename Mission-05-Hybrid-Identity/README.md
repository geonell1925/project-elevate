# Mission 5 – Hybrid Identity

## Overview

Mission 5 focused on understanding Hybrid Identity by exploring how on-premises Active Directory integrates with Microsoft Entra ID through Microsoft Entra Connect. The objective was to develop an Identity Engineer's troubleshooting methodology by investigating synchronization, authentication, Password Hash Synchronization (PHS), user lifecycle management, enterprise application integration, and security incidents.

Rather than concentrating on individual technologies, this mission emphasized understanding the complete identity lifecycle from user creation in Active Directory to authentication and authorization in Microsoft Entra ID and Microsoft 365.

---

## Mission Objectives

This mission was designed to strengthen my ability to:

- Understand Hybrid Identity architecture.
- Troubleshoot Microsoft Entra Connect synchronization.
- Differentiate object synchronization from password synchronization.
- Investigate Password Hash Synchronization (PHS).
- Troubleshoot enterprise application authentication.
- Analyze user lifecycle events such as onboarding, promotions, transfers, and offboarding.
- Apply structured identity troubleshooting using Microsoft Entra ID.

## Engineering Methodology

Every scenario followed the same engineering framework:

1. Assess business impact.
2. Determine the scope.
3. Eliminate healthy services.
4. Distinguish synchronization, authentication, and authorization.
5. Review recent identity lifecycle changes.
6. Gather evidence from logs and configuration.
7. Validate the root cause.
8. Apply the least disruptive resolution.
9. Verify successful authentication and authorization.
10. Document findings and lessons learned.

---

## Mission Outcome

Mission 5 strengthened my understanding of Hybrid Identity by teaching me how Active Directory, Microsoft Entra ID, Microsoft Entra Connect, and enterprise applications work together. More importantly, I learned to investigate identity problems using evidence rather than assumptions while considering both operational impact and security risk.
